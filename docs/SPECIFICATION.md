# 錬金術シミュレータ 仕様書 v1.0

## 1. 概要

予め用意された素材を合成し、アイテム棚とレシピ帳を充実させる錬金術シミュレータ。  
現状の「決まったレシピのみ成功・それ以外は失敗」という単調さを解消し、**AI活用**・**個人差**・**時間要素**・**明確な目的**を導入して、飽きずに楽しめる体験を実現する。

---

## 2. コア設計思想

| 項目 | 現状 | 改善後 |
|------|------|--------|
| 合成結果 | レシピ一致のみ成功、それ以外は失敗 | ほぼ常に何かが生まれる（AIで判定） |
| レシピ | 全員同じ固定レシピ | 個人ごとに異なる「あなただけのレシピ」 |
| 時間感覚 | なし | 「素早く」「普通」「ゆっくり」で結果に差 |
| 目的 | なし | 国一番の錬金術師を目指すランクアップ |
| ミッション | なし | 1日3つ・AI生成・達成判定もAI |
| セーブ | なし | Google Drive に個人フォルダで永続化 |
| 素材入手 | 最初から所持 | 市場でGを消費して購入 |

## 4. 機能仕様

### 4.1 ユーザー登録・工房

- **名前**と**工房名**を入力して登録
- Google Drive 上に `{名前}_{工房名}` のような**ユニークフォルダ**を自動作成
- このフォルダ内に以下を保存：
  - セーブデータ（JSON）
  - レシピ帳（個人レシピ）
  - 錬金履歴
  - 称号・ミッション進捗

---

### 4.2 AI錬金システム（OpenAI API）

**目的**: 固定レシピに依存せず、「何かが生まれる」体験を提供する。

#### 入力

- 素材A・素材Bの情報（名前・性質・既存DB）
  - **基本素材**に加え、**生成物も素材として使用可能**
  - 例：回復薬 + 薬草 → 上質回復薬、回復薬 + 宝石 → エリクサー など
- 調合ペース（素早く / 普通 / ゆっくり）
- 既に発見済みのレシピ（重複・矛盾を避けるため）

#### AIの役割

1. **結果生成**
   - 素材の組み合わせから、妥当な「錬金結果」を生成
   - 成功・失敗の二択ではなく、**常に何らかのアイテム・物質**を返す
   - 例：「薬草 + 水」→ 回復薬 / 濁った水 / 瘴気の雫 など

2. **品質・副産物**
   - 成功度（良質 / 普通 / 粗悪）を返す
   - 場合によっては副産物を1つ追加

3. **レシピ登録**
   - 初めての組み合わせで得られた結果を「あなただけのレシピ」として記録
   - 同じ組み合わせでも、**ユーザーごとに異なる結果**が出る可能性を許容（オプション）

#### プロンプト方針

- 錬金術の世界観・トーンを統一
- 結果はJSON形式で返す（例：`{ name, icon, quality, description }`）
- 不当に過強・過弱なアイテムを避けるガイドラインをプロンプトに含める

---

### 4.3 時間的要素（調合ペース）

| ペース | 効果 | 所要時間 |
|--------|------|----------|
| **素早く** | ランダム性↑、予期せぬ結果が出やすい | 短い（Tips短め） |
| **普通** | バランス型 | 標準 |
| **ゆっくり** | 安定傾向、高品質になりやすい | 長い（Tips長め） |

- プレイヤーが調合前に選択
- 時間差は主に**合成中の表示時間**と**AIへのヒント**として反映
- ランダム要素の強さをパラメータとしてAIに渡す
- 合成中は**アニメーションの代わりにTips**を表示（6.3参照）

---

### 4.4 「あなただけのレシピ帳」

- 固定の全員共通レシピではなく、**プレイした結果として発見したレシピ**のみが記録される
- 同じ「薬草+水」でも、ユーザーAは「回復薬」、ユーザーBは「瘴気の雫」というように差が出る
- レシピには「発見日」「使用回数」「成功率」などを付与可能
- 図鑑UIは「発見済み」と「未発見（？？？）」を区別して表示

---

### 4.4.1 合成チェーン（生成物の素材化）

**生成物も素材として使える**。錬金の結果は「売るだけ」「ミッション提出だけ」ではなく、**次の錬金の材料**になる。

#### 階層イメージ

```
[Tier0] 基本素材（市場で購入）
    ↓ 錬金
[Tier1] 初歩的な生成物（回復薬、毒薬、ガラス…）
    ↓ 錬金（生成物＋素材 or 生成物＋生成物）
[Tier2] 中級生成物（エリクサー、剣、天使…）
    ↓ 錬金
[Tier3] 上級・伝説級
```

- 基本素材同士 → Tier1
- 基本素材＋生成物、生成物＋生成物 → Tier2以降
- AIは素材の「段階」を考慮し、妥当な上位アイテムを生成

---

### 4.5 経済・市場システム（G）

**通貨**: **G**（ゴールド）。素材の購入・研究費・売却でやりくりする。

#### Gの入手

| 手段 | 内容 |
|------|------|
| **研究費** | 1日1回、ログイン時に一定のGを付与 |
| **ミッション達成** | ミッション完了時の報酬としてG |
| **生成物の売却** | アイテム棚の生成物を市場に売却してG化 |

#### 市場（マーケット）

- **市場インベントリ**から素材を購入
- **基本素材**：常時、すべての基本素材が並ぶ（Gを消費して購入）
- **ランダム素材**：日替わりで**3つ**の素材が追加表示（希少品や変わり種）
  - 毎日リセット時にAIまたは固定リストからランダム選出
  - 基本素材以外の素材・合成品が登場する場合あり

#### 素材の流れ

1. 研究費・売却・ミッションでGを稼ぐ
2. 市場でGを消費して基本素材を購入
3. 基本素材を錬金 → **Tier1生成物**
4. **生成物をさらに素材として錬金** → Tier2、Tier3…と上位へ連鎖
5. 生成物は売却するか、ミッション提出に使うか、**次の錬金の素材**として利用

※ 基本素材 → 生成物 → 上位生成物 の**合成チェーン**で深い錬金が可能。

※ 初期状態では、初回ログイン時の研究費で少しだけGを持ってスタートする想定。

### 4.6 目的：国一番の錬金術師（実績解除型）

**コンセプト**: 実績（称号）を解除して集め、一定個数でランクアップする方式。

#### 実績解除（称号獲得）

特定条件を満たすと**実績**が解除され、対応する**称号**を獲得する。

| 実績例 | 条件 | 獲得称号 |
|--------|------|----------|
| 初めての錬金 | 錬金を1回成功 | 釜に触れた者 |
| 駆け出し錬金術師 | 錬金を10回成功 | 駆け出し錬金術師 |
| 百錬の証 | 錬金を100回成功 | 百錬の錬金術師 |
| 初めてのレシピ | 新レシピを1つ発見 | 発見者 |
| 図鑑コレクター | レシピを10種類発見 | 図鑑マスター |
| 初ミッション | ミッションを1回達成 | 依頼請負人 |
| 三日連続 | ミッションを3日連続で達成 | 信頼の錬金術師 |
| 高品質職人 | 上質アイテムを10個作成 | 職人気質 |
| 伝説の一品 | 伝説品質を1つ作成 | 伝説の錬金師 |
| 素早さの極み | 「素早く」で50回調合 | 疾風の錬金術師 |
| 丁寧の証 | 「ゆっくり」で50回調合 | 慎重派錬金術師 |
| 合成の第一歩 | Tier2アイテムを初めて作成 | 上級錬金師見習い |
| 頂点へ | Tier3以上のアイテムを初めて作成 | 頂点を目指す者 |

※ 上記は一例。実装時に種類・条件・称号名を調整可能。

#### ランクアップ

実績を**一定個数**集めるとランクが上がる。

| ランク | 表示称号 | 必要実績数 |
|--------|----------|------------|
| 1 | みならい錬金術師 | 0（開始時） |
| 2 | 見習い錬金術師 | 3個 |
| 3 | 錬金術師 | 6個 |
| 4 | 熟練錬金術師 | 10個 |
| 5 | 名人錬金術師 | 15個 |
| 6 | 国一番の錬金術師 | 20個以上 |

- どの実績を解除するかはプレイヤーの自由
- 同じランク内で複数の実績を獲得していく達成感を重視

### 4.7 ミッションシステム

- **1日3つ**、日替わりでミッションを表示
- ミッション内容は**AI生成**（OpenAI API）
  - 例：「工房に届いた依頼：癒しの薬を作ってほしい」「町の祭り用の特別な炎を納品してほしい」
- プレイヤーが錬金し、**提出するアイテム**を選択して「ミッション達成」を申請
- **達成判定はAIが実施**
  - ミッション内容 + 提出アイテムの性質・説明 をAIに送り、適合するか判定
  - 適合度（完全一致 / 部分的 / 不適合）を返す

#### ミッションの種類例

- 依頼型：「〇〇を作ってほしい」
- 納品型：「△△を◯個納品」
- 発見型：「新しい〇〇系のアイテムを発見する」

### 4.8 保存・データ管理（Google Drive）

#### フォルダ構造例

```
Google Drive/
```

- ユーザー認証（Googleログイン）とフォルダ特定
- セーブの読み書き（Drive API）
- OpenAI API 呼び出し（ミッション生成・錬金結果・達成判定）
- 日付管理（「今日のミッション」の切り替え）

## 5. 追加提案：より面白くする要素

### 5.1 工房のアップグレード

- 錬金回数やランクに応じて**工房レベル**が上昇
- レベルアップで：
  - 新しい基本素材が解放される
  - 3素材同時調合が可能になる
  - 錬金釜の見た目が変わる

### 5.2 失敗のバリエーション

- 単なる「何も起きない」ではなく：
  - **微妙な結果**：使えないが何かは残る
  - **副産物**：狙いと違うが価値のあるものが混ざる
  - **大失敗**：材料の一部を失う（稀）

### 5.3 顧客・町の人

- ランダムで「町の人」が依頼を持ってくる
- ミッションとは別に、短いクエストとして実装
- 報酬として希少素材や特別レシピのヒント

### 5.4 錬金日記（AI要約）

- 一定期間の錬金履歴をAIが要約
- 「先週は回復薬を多く作り、新しい雷のレシピを発見した」のような一文を生成
- 思い出として閲覧可能

### 5.5 品質・希少度

- 同じアイテムでも**品質**（粗悪 / 普通 / 上質 / 伝説）を持つ
- 品質が価値・ミッション適合度に影響
- 伝説品質は特別な演出

### 5.6 ひらめき・AI提案

- 失敗した組み合わせに対して、AIが「この組み合わせなら〇〇が作れそう」と提案
- プレイヤーの探索を後押しする仕組み

### 5.7 実績・ランクの演出

- 実績解除時にポップアップで祝福メッセージ
- ランクアップ時に短いストーリーや祝福メッセージ
- 実績・称号に応じたバッジ・アイコン表示

### 6.3 導線設計（合成チェーンを分かりやすく）

ユーザーが「生成物も素材になる」と気づき、自然に上位合成へ進めるための導線。

#### 所持アイテムの一元化

| エリア | 内容 | 用途 |
|--------|------|------|
| **所持アイテム** | 購入した基本素材 ＋ 錬金で得た生成物を**同一エリア**に表示 | どちらも釜へドラッグして錬金に使用可能 |

- 素材ボックスとアイテム棚を統合、または**並べて表示**して「どちらも釜に入れられる」ことが一目で分かるようにする
- タブやフィルタで「素材のみ」「生成物のみ」を切り替え可能（オプション）

#### 視覚的ガイド

- **Tierバッジ**：各アイテムに Tier1 / Tier2 / Tier3 の小さな表示
  - 「このアイテムはまだ合成に使える」ことが伝わる
- **錬金可能アイコン**：生成物に「🔬 素材に使える」などのマークを表示
- **初回ツールチップ**：「作ったアイテムも釜にドラッグして、さらに上質なものを目指せます」

#### 進行ヒント

- **ひらめき提案**（5.6）：未発見レシピに対して「〇〇と△△を組み合わせてみる？」とAIが提案
- **レシピ帳連携**：発見済みレシピの材料が所持アイテムにあれば「作成可能」バッジを表示
- **ミッション連携**：ミッション達成に近いアイテムを持っていれば「提出候補」としてハイライト

#### チュートリアル導線（初回プレイ）

1. 基本素材を2つ釜に入れて調合
2. 生成物が釜に出現 → 「アイテム棚に追加されました」
3. **「このアイテムは、もう一度釜に入れるとさらに別のものが作れるかもしれません」** と案内
4. アイテム棚の生成物を釜へドラッグできることを示す

#### 合成中のTips表示（アニメーションの代わり）

合成処理中は**アニメーションの代わりにTips**を表示する。融合演出などは行わず、合成ヒント・操作マニュアルなどを表示して結果が出るまで待つ。

**表示内容の例**

| カテゴリ | 例 |
|----------|-----|
| **合成ヒント** | 「生成物も釜に入れると、さらに上位のアイテムが作れます」「同じ系統の素材を組み合わせると新しい発見があるかも」 |
| **操作マニュアル** | 「素材はドラッグ＆ドロップで釜に入れられます」「アイテム棚のアイテムをクリックすると売却できます」 |
| **調合ペースの説明** | 「素早く=予想外の結果、ゆっくり=安定志向。状況に応じて使い分けよう」 |
| **システムTips** | 「ミッション達成でGと実績が手に入ります」「市場の日替わり素材は毎日変わります」 |
| **世界観** | 「工房で国一番の錬金術師を目指そう」 |

- ペースに応じた表示時間：素早く=短いTips、ゆっくり=長めのTips or 複数表示
- ランダム or 未読優先で表示し、繰り返しプレイでも飽きにくくする
- 釜エリア全体または中央にTipsを表示。結果発表時にスッと切り替わる

## 8. セキュリティ・運用

### 8.1 APIキー

- OpenAI APIキーは**GAS側のみ**で保持
- フロントからは一切APIキーを露出させない

### 8.2 レート制限

- 1ユーザーあたりの錬金回数・API呼び出しに上限を設ける（コスト対策）
- 無料枠を考慮した1日あたりの回数制限

### 8.3 データ整合性

- セーブはGASで検証し、不正な書き換えを防ぐ

## 10. 用語集

| 用語 | 説明 |
|------|------|
| 錬金 | 2つの素材（基本素材 or 生成物）を組み合わせて新しいアイテムを作ること |
| 合成チェーン | 基本素材→Tier1生成物→Tier2…と、生成物を素材として上位アイテムを作る流れ |
| Tier | 生成物の段階。Tier1=初級、Tier2=中級、Tier3=上級・伝説級 |
| 調合ペース | 素早く / 普通 / ゆっくり。結果のランダム性・品質に影響 |
| Tips | 合成中の表示。アニメーションの代わりに合成ヒント・操作マニュアル等を表示 |
| あなただけのレシピ | プレイ中に発見した組み合わせと結果の記録 |
| ミッション | 1日3つ、AI生成の依頼。達成はAIが判定 |
| G | 通貨。研究費・ミッション・売却で入手。市場での素材購入に消費 |
| 市場 | 基本素材＋日替わり3種を購入、生成物を売却する場所 |
| 研究費 | 1日1回ログイン時に付与されるG |
| 実績 | 特定条件達成で解除される称号。例：錬金10回成功→駆け出し錬金術師 |
| 称号（ランク） | 実績を一定数集めるとランクアップ。みならい〜国一番の錬金術師 |
