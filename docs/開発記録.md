# 錬金術シミュレータ 開発報告書（現行仕様統合版）

## 1. 本書の要約

本プロジェクトは、固定レシピ中心の合成ゲームを、AI活用による「発見」と「接客運営」が循環する体験へ再設計した。  
現在の主導線は、`錬金 → 商品陳列 → 来客購入 → 売上獲得` であり、プレイヤーは短時間でも進捗実感を得られる。  
本書は、提出済み改善記録と現行実装のみを根拠として、対外説明向けに仕様と改善経緯を整理したものである。

### 主要ハイライト

| 項目 | 現在の状態 |
|------|------------|
| コア体験 | 錬金と店舗運営を1画面で循環 |
| 操作負荷 | 画面遷移を抑え、次操作を視覚誘導 |
| 素材導線 | 基本素材は無限、作成物は再錬金・販売に活用 |
| 収益導線 | 顧客の自動購買により売上（G）を継続獲得 |
| 世界観演出 | 案内役ピペット、来客コメント、通知演出を統合 |

## 3. プロジェクト背景

### 3.1 初期段階で顕在化した課題

- 固定レシピ不一致時の失敗比率が高く、探索の継続性が低い

### 3.2 プロトタイプ段階で顕在化した課題

- 素材購入・ミッション達成を含む導線が複雑で、目的が分かりにくい
- 表記と画面導線に学習コストがあり、ライト層が離脱しやすい

これに対し、開発方針を「複雑な管理型」から「直感的な循環型」へ転換した。

---

## 4. 改善の全体像

### 4.1 コンセプト転換（Before / After）

| 観点 | Before | After |
|------|--------|-------|
| 合成結果 | レシピ一致のみ成功 | AI判定で常に結果を返す |
| 基本ループ | 購入→錬金→提出 | 錬金→陳列→自動販売 |
| 主な達成感 | 条件達成中心 | 発見と売上の蓄積 |
| 操作導線 | 画面移動が多い | 1画面中心で完結 |
| 世界観提示 | 説明寄り | キャラクターと通知で補助 |

### 4.2 段階別の課題解消

| フェーズ | 主な課題 | 実施した改善 |
|----------|----------|--------------|
| プロトタイプ | 進行が煩雑、用語が難解 | 基本素材無限化、導線整理、ピペット導入 |
| 実装統合（中間工程） | 改善案の要素が分散し、遊びの流れが分断されやすい | メイン画面集約、在庫/陳列/接客の連携実装、売上ログと演出の接続 |
| 試験版 | 次操作が不明瞭、状態変化が見えない | ボタン強調、表記統一、トースト通知、自動スクロール |

### 4.3 プロトタイプ版と試験版の間の開発工程（実装統合）

試験版に入る前段として、改善アイデアを実際のプレイ導線に落とし込む「実装統合フェーズ」を実施した。  
この工程の目的は、個別改善を単発で適用するのではなく、`錬金 → 陳列 → 接客 → 売上` の循環を一連の体験として成立させることにある。

主な実施内容:

- メイン画面を中心に、素材置き場・釜・商品だな・来客導線を同時に扱える構成へ統合
- 作成物を「再錬金素材」と「販売商品」の両方で使える在庫ルールを整理
- 開店/閉店の状態管理、来客キュー処理、自動購入判定の実装
- 売上加算、ヘッダー表示、日次売上台帳を接続し、運営結果を可視化
- ピペット演出、顧客コメント、通知表示を加え、状態変化の理解を補助

この中間工程により、試験版では「分かりやすさ」と「操作誘導」の最終調整に集中できる開発段階へ移行した。

## 5. 現在の仕様（実装準拠）

### 5.1 体験フロー

```text
素材投入
  ↓
錬金（2素材 + ペース + 個数）
  ↓
作成物を商品だなへ配置
  ↓
来客が希望カテゴリ商品を自動購入
  ↓
売上Gを獲得し、次の錬金へ
```

### 5.2 機能一覧（現行）

| 機能領域 | 現在の仕様 |
|----------|------------|
| 錬金 | 2素材合成、ペース3種、個数1〜20、Tips表示 |
| 素材管理 | 基本素材は無限、作成物は在庫管理 |
| 商品だな | 作成物を陳列し、販売対象化 |
| 来客システム | 希望カテゴリ・予算・気質を持つ顧客が来店 |
| 販売処理 | 条件一致で自動購入、不一致で離脱 |
| ゴールド | 売上加算、ヘッダー表示、増減演出 |
| 売上記録 | 日次台帳（件数・カテゴリ・金額・時刻） |
| レシピ | 発見記録、利用回数更新、レシピブック表示 |
| 実績・ランク | 条件同期、解除トースト、称号表示 |
| 保存 | API接続時はGAS保存、未接続時はローカル動作 |

### 5.3 UI/UX上の特徴

- 次に押すべき操作を強調し、迷いを減らす
- 購買/離脱をトーストで即時通知する
- ピペットが説明役として文脈を補完する
- 顧客コメントを表示し、結果の意味を伝える
- モバイルでも成立するレイアウトを維持する
